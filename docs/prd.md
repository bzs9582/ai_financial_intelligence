# 产品需求文档

这个文件定义第一阶段 MVP 的真实边界。为了让 Codex 能顺利启动开发，第一版只做“情报聚合 + 多智能体报告输出”的核心闭环，不在首轮加入复杂社区和结算模块。

## 产品目标

让用户在单一界面中，针对指定资产和时间窗口，快速获得一份由多智能体协作生成的结构化投研报告，报告同时覆盖：

- 客观数据异动
- 宏观背景解释
- 短期结构与流动性视角
- 风控反驳与失效条件

## MVP 功能范围

- 用户可以选择一个资产标的和分析时间窗口，手动发起一次分析
- 系统可以通过代理抓取基础行情和部分宏观数据
- 系统可以将原始数据转为统一的结构化 JSON 情报
- 系统可以调用四个角色的多智能体链路，生成统一格式报告
- 用户可以在页面查看最近一次分析报告与关键信号

## 第一阶段核心报告字段

- 标的与时间窗口
- 数据快照摘要
- Peter 的量化/数据结论
- Venturus 的宏观结论
- Pivot 的结构与流动性结论
- Reality Check 的反驳与风控结论
- 多头逻辑
- 空头逻辑
- 情景概率分布
- 失效条件
- 风险提示

## 核心用户流程

1. 用户打开 Web 页面
2. 用户选择资产，例如 BTCUSDT
3. 用户选择分析周期，例如 4h 或 1d
4. 用户点击“开始分析”
5. 系统通过代理获取最新数据并做标准化处理
6. 多智能体生成结构化报告
7. 页面展示完整报告与最近一次更新时间

## 技术边界

- 前端技术：首版采用服务端渲染页面，优先简单稳定，不先上复杂前端框架
- 后端技术：Python 3.12 + FastAPI
- 数据请求：优先使用 `httpx`，统一支持代理环境，默认代理地址为 `http://127.0.0.1:4780`
- 数据库方案：SQLite，用于缓存情报快照和保存最近报告
- 智能体编排：首版优先采用自定义轻量编排层；LangGraph 或 CrewAI 作为后续可选增强，不作为 MVP 强依赖
- 指标计算：使用自定义轻量计算模块，不引入 `pandas_ta`
- 登录方案：MVP 不做用户登录
- 部署方式：先保证本地可运行，后续再扩展到云端部署

## 数据源范围

MVP 首轮至少覆盖：

- Binance 行情与基础衍生品数据
- FRED 中至少一个宏观指标
- 一个可替换的文本事件源接口

如果外部 API 密钥不完整，开发态允许先用 fixtures 或模拟数据保持流程可跑通。

## 项目约束

- 开发速度要求：优先快速完成可运行原型
- 预算要求：尽量减少第三方依赖和付费服务数量
- 合规要求：仅提供研究与策略参考，不提供投资建议或自动交易执行
- 目标设备：桌面浏览器优先

## 当前不做

- 声誉广场正式版
- 智能积分托管与策略结算
- 自动下单
- 社区社交系统
- 多用户权限体系
- 支持过多资产类别和复杂因子库
