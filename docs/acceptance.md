# 验收标准

这个文件定义第一阶段 MVP 的完成标准。

## 必须满足

- [x] 项目可以通过文档说明在本地启动
- [x] Web 页面可以手动发起一次分析
- [x] 系统可以根据代理配置发出外部数据请求
- [x] 系统可以把采集到的数据转换为统一结构化 JSON
- [x] 系统可以生成一份包含多空逻辑、概率分布和失效条件的结构化报告
- [x] 至少有一条自动化测试覆盖“发起分析并得到报告”的关键路径
- [x] 指标计算模块不依赖 `pandas_ta`
- [x] 关键失败信息可以在日志或界面中定位

## 最好满足

- [x] 页面提供最近一次报告摘要
- [x] 提供开发态 mock 数据模式，方便在无外部 API 情况下调试
- [x] 提供至少一个宏观指标缓存
- [x] 提供基础响应式布局

## 完成定义

- [x] 文档与当前实现一致
- [x] `factory.toml` 中定义的验证命令全部通过
- [x] `docs/tasks.md` 中首批 MVP 任务全部完成或已合理拆分
- [x] 输出内容明确标注“仅供研究参考，不构成投资建议”
